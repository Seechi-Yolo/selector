# 产品需求：使用教程与沙箱

## 交付范围

1. 首次使用：在用户通过**工具栏扩展动作或快捷键**打开选取（主内容脚本已注入）之后，若尚未关闭过引导，则展示**轻量弹窗**（小型卡片，非多步向导）；**不得**在仅打开普通网页、未激活选取时自动弹出。弹窗提供主操作「打开使用教程」，在新标签页打开扩展内顶栏页并默认激活「使用教程」标签；提供「跳过」关闭且在整个扩展安装周期内跨站点不再展示。文案与主路径事实须与 [../product/usage-tutorial.md](../product/usage-tutorial.md) 一致。实现位于 `features/editor-onboarding` 及选取会话挂载逻辑（与 `assets/content.js` 同源注入）；存储键名由代码维护，本需求文件不规定。
2. 扩展上下文菜单：提供唯一菜单项，用户选择后打开单一扩展内页面，该页面 URL 形如 `chrome-extension://` 加扩展标识及 HTML 资源路径。
3. 上述扩展内页面包含两个标签页，标题分别为「使用教程」与「沙箱」，用户可在其间切换。标签切换控件置于**页面顶部**，作为全局 Tab 栏；**不得**将「顶 Tab + 下方整块内容上下堆叠」作为唯一主结构，以免目录、正文、沙箱与说明挤在同一纵轴上、层次不清。
4. **页面信息架构（主体布局）**：在顶部 Tab 之下，当前标签对应的主体区域固定为**左右分栏**：
   - **左侧**：目录区（大纲 / 锚点导航），随 Tab 切换展示「使用教程」或「沙箱」各自的章节或模块目录，用户可快速跳转。
   - **右侧**：该 Tab 下的主内容区。
5. 使用教程标签页以静态文本与图像呈现内容。图像占位与文件命名须符合 [../product/usage-tutorial.md](../product/usage-tutorial.md)。章节顺序与该文档「主路径与阅读顺序」所列七步一致；任何偏离须在该文档内修订并保留版本说明。教程正文与配图出现在**右侧主内容区**；左侧目录与正文结构对应。
6. **沙箱标签页的主内容区（右侧）**再分为**左右两栏**（内层左右，与全页「左目录、右内容」叠加）：
   - **内层左侧**：沙箱容器，渲染与当前注入式用户界面相关的组件与样式，支持点击与简单操作的交互反馈。
   - **内层右侧**：以**文字**为主的引导与反馈区（可含操作日志、步骤说明、点击后的效果说明等），让用户在操作沙箱的同时**并排**阅读「点了会怎样」，避免将沙箱与说明纵向上下叠放导致视线来回跨越整块高度。  
   沙箱展示范围限于当前仓库已实现之面板、叠加层、列表与按钮等代表状态，不包含里程碑四中尚未实现的「从站点回写样式至提示词」完整产品链路。
7. 教程弹窗与教程标签页全文在事实描述上不得相互矛盾。弹窗仅作跳转引导时，须以明确主操作打开扩展内页面并激活「使用教程」标签页。

## 验收条件

扩展上下文菜单中存在且仅存在一条入口，指向包含双标签页的上述扩展内页面，两个标签页均可访问且状态可切换。

扩展内页面布局符合交付范围所述信息架构：顶部为「使用教程 / 沙箱」Tab；Tab 下方主体为**左目录、右内容**；沙箱 Tab 下右侧主内容区内为**左沙箱容器、右文字引导/日志**，沙箱与引导区非仅靠上下堆叠作为唯一呈现方式。

使用教程标签页章节结构与 `docs/product/usage-tutorial.md` 所载主路径一致，或该文档已同步更新以反映实现。

沙箱标签页内至少三类控件在用户操作下产生可见状态变化，例如悬停、激活或展开收起。上述体验不依赖向任意普通网页注入内容脚本。用户操作后，引导/反馈区能体现与操作相关的文字说明或日志（可与具体控件一一对应，实现细节由工程说明补充）。

在全新浏览器配置文件或等价于首次安装扩展的环境中，自菜单进入扩展页、依次访问教程标签页与沙箱标签页并返回教程标签页，浏览器控制台不得出现导致功能中断的未捕获错误。

扩展上下文菜单的物理入口位置须在 [../engineering/extension.md](../engineering/extension.md) 或仓库根目录 README 之一中写明，便于用户查找。

本需求所涉菜单项文案、目标 URL 及与独立教程页、独立沙箱页构建产物之间的关系须在 `extension.md` 中维护为当前事实。
