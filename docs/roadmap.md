# 实施顺序

## 产品里程碑

| # | 状态 | 名称 | 一句话 |
|---|:---:|------|--------|
| 1 | ✅ | **重构与浏览器扩展** | DDD 拆分 + MV3 扩展；与 README 行为一致。（**阶段一已完成**） |
| 2 | 待办 | **选取范围** | 复杂页面上尽量能指到真正要改的区域，减少「看得到选不中」。 |
| 3 | 待办 | **设置与提示词模板** | 默认复制结构可改、段落可关、可存多套，不必所有人同一套拼法。 |
| 4 | 待办 | **样式与高级提示词** | 对照观感、在工具里试调样式，导出信息更完整、更利于 AI 改界面。 |
| 5 | 待办 | **设计系统** | 有团队规范时，可从规范选用或对齐，再带入给 AI 的提示。 |

**远期**：用户手建 **交互流图** → 生成提示词（不自动录制真实操作链路）。

各里程碑的详细验收与待办方向见 [capability-backlog.md](./capability-backlog.md)（里程碑 **1** 已压缩为「已完成」摘要）。
