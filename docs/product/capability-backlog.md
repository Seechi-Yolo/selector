# 里程碑待办

本文件与 [roadmap.md](./roadmap.md) 中的里程碑编号一致。架构约定见 [../engineering/architecture-ddd.md](../engineering/architecture-ddd.md)，扩展运行时见 [../engineering/extension.md](../engineering/extension.md)。

## 里程碑一：重构与浏览器扩展

状态为已完成。源代码按领域驱动设计组织于 `src/` 目录。执行 `npm run build` 生成 `dist/` 目录，以加载未打包扩展的方式安装。用户主路径为指认元素、撰写说明、复制提示词，与仓库根目录 README 描述一致。新增需求归入里程碑二及以后条目。

## 里程碑二：选取范围

交付内容：在含画布与多层 DOM 结构的常见复杂页面上，用户能够稳定将选取目标对准意图修改的区域。

验收标准：在上述页面类型上，用户能够连续完成指认、说明与复制整条路径；选取结果与操作意图一致。

## 里程碑三：设置与提示词模板

交付内容：提供专用设置界面；默认复制结构支持编辑；各导出段落支持开关；支持保存多套常用模板。

验收标准：设置界面展示当前复制结果的预览；用户修改模板或开关后再次复制，剪贴板内容与设置一致。

## 里程碑四：样式与高级提示词

交付内容：支持对照当前页面观感；在扩展内对颜色与间距等视觉属性进行试调；导出结构较里程碑三更为完整。

验收标准：用户完成选中、对照或微调观感、触发复制后，导出文本较前一阶段更便于向人工智能描述目标界面形态。

## 里程碑五：设计系统

交付内容：当组织具备统一设计规范时，用户可从规范中选用或对齐相关项，并将其纳入提示词。规范导入方式在里程碑五详细设计阶段确定并写入文档。

验收标准：具备设计规范的用户能够完成选用或对齐并写入提示；不具备设计规范的用户仍仅依赖里程碑四所定义能力即可完成主要任务。

## 远期条目

交付内容：用户通过手绘步骤图表达操作先后与参与元素，结合当前选中项与文字说明生成提示词。不承诺自动录制网站内真实发生过的事件序列。

验收标准：用户能够使用流程图加文字得到可直接用于人工智能协作的描述文本，该描述不要求与后台真实数据流一致。

## 上架与说明

交付内容：应用商店展示文案、截图与隐私说明；版本更新说明。

验收标准：新用户能够依据商店页判断安装价值与安装后能力边界；隐私承诺与扩展实际行为一致。
