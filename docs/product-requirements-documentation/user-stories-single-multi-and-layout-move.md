## 故事一：只改一个页面元素

**角色**：在真实页面上改 UI 或排版的编辑者（或配合 AI 写代码的开发者）。

**场景**：页面上只有一个区块需要调整，不需要多选或整段范围说明。

**目标**：我能**点选该元素**，在说明关状态下按 **Enter** 打开画布下的 **「修改说明」**，写清这一项要怎么改；变更会参与 **复制提示词** 的拼装（含防抖自动写剪贴板与提交时再刷新），以便我把**单目标、单意图**交给下游 AI 或同事。

**验收要点（故事级）**：

- 选取建立后默认不弹输入层；**Enter** 后出现该项下的 **修改说明** 浮层（与 **I-28**、**I-22** 一致）。
- **复制提示词** 中能唯一定位到该元素（选择器、上下文等），并带上该项 **修改说明** 正文。

---

## 故事二：同时对多个已选元素下「修改指令」

**角色**：一次改一批相关控件的编辑者。

**场景**：用 **范围选取** 或 **Shift+点击** 选中多个元素；需要先交代这批选取**共同**要达成什么，再对其中若干项写**各自**的修改说明。

**目标**：多选形成结束后，我能先按 **Enter** 打开 **「对当前选取的说明」**（整体层），写完并提交放行后，再针对焦点项或逐项入口写 **「修改说明」**；最终 **复制提示词** 里**整体块在逐项之前**（与 **I-13**、**I-14** 线性默认一致），以便 AI 同时看到**整批意图**和**各元素差异**。

**验收要点（故事级）**：

- **≥2** 项完成后先处于说明关 **N**，**Enter** 进入整体说明 **O**；整体放行后才进入逐项路径。
- 导出文稿同时包含可选的 **对当前选取的说明** 与各元素的 **修改说明**（无冗余项时可省略对应段落，口径以 **I-14** 为准）。

---

## 故事三：把某个按钮移动到另一容器的「上方」或指定参考位置

**角色**：调整布局或信息架构的产品 / 前端使用者。

**场景**：页面上有一个**待移动的按钮**（或同类控件），以及一个**目标容器**或**可见的参考元素**（例如「放到侧栏容器顶」「相对某标题上方留白对齐」）。

**目标**：通过选取与说明，让 **复制提示词** 里同时带上**被移动控件**与**目标/参考位置**的 DOM 与文案上下文，并在 **修改说明** 或（多选时）**对当前选取的说明** 中写清**空间关系**（例如：移动到某容器上方、与某元素对齐、插入到某列表项之前等），以便接收方 AI 能依据结构化上下文执行布局或代码改动，而不是靠模糊的一句「挪一下」。

**验收要点（故事级）**：

- 用户至少能**点选或框选**到「被移动项」与「目标容器/参考项」中的相关节点（必要时配合 **Shift+点击** 多选）；说明文字明确写出**相对谁**、**哪一侧/哪一轴**等关系（由用户撰写，产品不强制模板句式）。
- **复制提示词** 对多选场景包含各选中项的元数据；若使用整体说明，整段意图与逐项说明在文稿中的位置关系符合 **I-14** 默认顺序。
- Slash 落地后，**移动** 动作与 **参考点选** 路径须满足 [`selection-session-slash-and-multi-step.md`](./selection-session-slash-and-multi-step.md) **S-03**、**S-04**。

## 故事四：选中页上按钮并套用设计系统中的某一按钮样式后复制提示词

**角色**：希望宿主页上的按钮与设计系统（或团队组件库）里已有变体对齐的设计师 / 前端。

**场景**：页面上有一个**真实按钮**（或表现为按钮的控件）需要改版；团队在设计系统文档或 Storybook 等渠道里已经定义了多种**按钮样式**（如 primary / secondary / ghost / danger 等命名与视觉规范）。

**目标**：我能**点选该按钮**，在扩展内（或选取会话相关 UI 中）**浏览并选定**某一设计系统按钮样式；系统将「当前 DOM 上下文 + 所选样式名称 / 令牌 / 规范摘要」一并纳入 **复制提示词**，以便下游 AI 或开发者按同一套设计语言改代码，而不是只在 **修改说明** 里手写「做成主按钮」这类易歧义句子。

**验收要点（故事级）**：

- 选取目标按钮后，用户有一条清晰路径完成「**选样式 → 写或补全说明（可选）→ 复制提示词**」；样式选择在导出文稿中有**可机读或可复述**的表述（具体字段名与 UI 形态由实现与路线图约定；Slash 见 **S-02**）。
- **复制提示词** 仍须包含该按钮的定位与上下文（与故事一一致）；若用户额外写了 **修改说明**，与样式信息**不冲突**或可合并理解。
