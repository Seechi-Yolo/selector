# 产品需求：使用教程与沙箱

## 交付范围

1. 首次使用：在用户首次使用扩展时展示教程弹窗。实现位于 `features/editor-onboarding` 及相关应用层挂载逻辑。首次展示是否已读及重复展示策略由实现写入工程说明，本需求文件不规定存储键名。
2. 扩展上下文菜单：提供唯一菜单项，用户选择后打开单一扩展内页面，该页面 URL 形如 `chrome-extension://` 加扩展标识及 HTML 资源路径。
3. 上述扩展内页面包含两个标签页，标题分别为「使用教程」与「沙箱」，用户可在其间切换。
4. 使用教程标签页以静态文本与图像呈现内容。图像占位与文件命名须符合 [../user/user-tutorial.md](../user/user-tutorial.md)。章节顺序与该文档第二节「主路径与阅读顺序」所列七步一致；任何偏离须在 `user-tutorial.md` 内修订并保留版本说明。
5. 沙箱标签页渲染与当前注入式用户界面相关的组件与样式，并支持用户通过点击与简单操作获得交互反馈，用于团队调试样式及用户无风险体验。沙箱展示范围限于当前仓库已实现之面板、叠加层、列表与按钮等代表状态，不包含里程碑四中尚未实现的「从站点回写样式至提示词」完整产品链路。
6. 教程弹窗与教程标签页全文在事实描述上不得相互矛盾。当弹窗仅承载最短路径说明时，弹窗内须提供明确控件，导航至本扩展内页面并激活使用教程标签页。

## 验收条件

扩展上下文菜单中存在且仅存在一条入口，指向包含双标签页的上述扩展内页面，两个标签页均可访问且状态可切换。

使用教程标签页章节结构与 `docs/user/user-tutorial.md` 所载主路径一致，或该文档已同步更新以反映实现。

沙箱标签页内至少三类控件在用户操作下产生可见状态变化，例如悬停、激活或展开收起。上述体验不依赖向任意普通网页注入内容脚本。

在全新浏览器配置文件或等价于首次安装扩展的环境中，自菜单进入扩展页、依次访问教程标签页与沙箱标签页并返回教程标签页，浏览器控制台不得出现导致功能中断的未捕获错误。

扩展上下文菜单的物理入口位置须在 [../engineering/extension.md](../engineering/extension.md) 或仓库根目录 README 之一中写明，便于用户查找。

本需求所涉菜单项文案、目标 URL 及与独立教程页、独立沙箱页构建产物之间的关系须在 `extension.md` 中维护为当前事实。
