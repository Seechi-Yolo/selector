# PRD：扩展内使用教程与沙箱

**里程碑**：里程碑一（扩展与教程入口）。事实口径：`README.md`、`manifest.json`。

## 本仓库 PRD 体例（摘要）

一条 PRD 只写**要做什么**：短句、可验收；同一事实只写一次。实现细节不在 PRD 展开。推荐块：**要做什么** → **用户可感知行为** → **内容与素材** → **验收** → **路线图（一行）**。

---

## 1. 要做什么

- 用户进入选取会话后，可看到**轻量引导**（小卡片）；可打开扩展内「教程与沙箱」顶栏页或跳过；关闭状态跨站点持久。
- **顶栏页**经扩展菜单**唯一**入口打开；含「使用教程」「沙箱」两标签，顶部切换。
- **布局**：Tab 下**左目录、右主内容**；教程侧为静态文图；沙箱侧主内容为**左沙箱、右说明**并排。
- **沙箱**：呈现与注入式界面一致的代表状态（面板、叠加层、列表、按钮等），可有简单操作反馈；右侧说明区可写步骤或操作结果。

---

## 2. 用户可感知行为

1. 进入选取后，可出现引导；可打开教程页或跳过。  
2. 从菜单进入顶栏页，在教程与沙箱之间切换。  
3. 教程页：左侧目录跳转，右侧按主路径阅读并看图。  
4. 沙箱页：左侧操作示例，右侧阅读说明；操作后说明区有对应文字。  
5. 快捷键与最短路径与 `README.md`、`manifest.json` 中 `commands` 一致。

---

## 3. 内容与素材（教程对齐）

**主路径（顺序固定）**：① 启动扩展 ② 主面板总览 ③ 页内选取 ④ 选中叠加层 ⑤ 批注 ⑥ 复制提示词 ⑦ 空格暂停、撤销、清空、关闭会话。

**占位与动图**：未录制时使用 `assets/tutorial/placeholder.gif`（须在 `web_accessible_resources` 声明）。建议文件名：`01-open-selector` … `14-esc-undo`（05、13 可选；14 可与 10–12 合并）。

**体例**：每节含标题、一句主题、若干步骤、图；教程单页纵向滚动，首屏可锚点到启动 / 选取 / 复制。

---

## 4. 验收

- [ ] 菜单有且仅有一条入口，打开含两标签的扩展内页；两标签可切换、可访问。  
- [ ] 顶 Tab；其下左目录右主内容；沙箱 Tab 内左沙箱、右说明并排。  
- [ ] 教程章节顺序与 §3 主路径一致。  
- [ ] 沙箱内至少三类控件在操作下有可见状态变化；说明区有与操作相关的文字。  
- [ ] 首次安装等价场景下，进入该页并切换标签，功能可用。  
- [ ] 菜单入口在根目录 `README.md` 可找到；菜单文案与 URL 与 `README.md`、`manifest.json` 一致。
